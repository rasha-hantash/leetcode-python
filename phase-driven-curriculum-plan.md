# Phase-Driven Curriculum — Replace the 90-Day Prescription with Phase Budgets

## Why

`prep-plan-daily.md` is currently a 1020-line Day-by-Day prescription that the engine never enforces — it just walks the flat problem list in document order. The Day-N annotations are decorative. **Phases are the actual schedule.**

Refactoring to a phase-driven model:

- Drops ~770 lines of dead prose from `prep-plan-daily.md`
- Makes the engine's behavior match its docs (currently the docs claim a Day-by-Day prescription that the engine ignores)
- Lets each phase declare its pattern scope, daily new-problem budget, and difficulty cap
- Removes calendar pressure when life happens (phase advancement is ledger-driven, not date-pinned)

## Target Architecture

`patterns: null` = any pattern. `difficulty_cap: M` = no Hards. `difficulty_floor: H` = only Hards. `new_per_day: 0` = recall-only mode.

### Phase advancement (ledger-driven)

Current phase = the lowest-numbered phase whose problem set has untouched entries (respecting `difficulty_cap` / `difficulty_floor`). When all of a phase's eligible problems are touched ≥1, advance to the next.

Edge case: Phase 8 (Reinforcement) is reached when every phase before it is exhausted. `new_per_day=0` means today.md only shows Recall.

### Engine behavior changes

`compute_new(curriculum, ledger, phases, today)`:

1. Determine current phase via the rule above.
2. Filter curriculum to problems whose pattern is in `phase.patterns` (or all if `null`) AND `difficulty <= phase.difficulty_cap` AND `difficulty >= phase.difficulty_floor`.
3. Take the first `phase.new_per_day` untouched, ordered by source rank → difficulty rank → document order (existing logic).

`today.md` header changes:

- "Day N" stays (count of distinct ledger dates), but framed as a calendar reference, not a prescription.
- New line: phase name + budget. Example: `Phase 1 — Linear Patterns E+M · 5 new/day budget`.
- New section above Recall: today's time blocks (mock-aware). Example:
  ```
  ## Time blocks
  - 9:00–13:00  Recall
  - 14:00–15:30 System Design
  - 15:30–19:30 DSA New
  ```
  On mock days: `14:00–16:00 Mock` + `16:00–17:30 SD` + `17:30–19:30 DSA New`.
